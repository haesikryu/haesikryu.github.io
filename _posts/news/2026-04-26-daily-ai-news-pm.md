## 1. AI가 해결한 수십 년 된 수학 난제: 에르되시(Erdős) 문제 풀이

**요약:** 아마추어 수학자인 리암 프라이스(Liam Price)가 GPT-5.4 Pro를 활용하여 오랜 수학적 난제 중 하나인 '에르되시 추측(Erdős conjecture)'의 일부를 해결했다는 소식이 전해졌습니다. 구체적으로는 '프리미티브 집합(primitive set)'의 에르되시 합(Erdős sum) 최솟값이 1이며, 원소들이 무한대로 갈수록 그 값에 수렴한다는 문제입니다. 리암 프라이스는 단 한 번의 프롬프트로 이 해법을 얻어냈으며, 이를 수학계의 난제 공유 사이트인 erdosproblems.com에 게시했습니다. 이후 전문가인 케빈 포드(Kevin Ford) 등에 의해 이 증명이 타당한 것으로 받아들여지고 있습니다.

**의미:** 이번 사건은 AI가 단순히 텍스트를 생성하는 수준을 넘어, 인간이 수십 년간 해결하지 못한 고도의 추론과 창의성이 필요한 수학적 증명 영역까지 도달했음을 시사합니다. 특히 '아마추어'가 AI를 도구로 사용하여 학술적 성과를 냈다는 점은, AI가 전문 지식의 장벽을 낮추고 과학적 발견의 속도를 비약적으로 높일 수 있는 강력한 '지능 증폭기' 역할을 하고 있음을 증명합니다.

## 2. Claude Code와 MCP를 이용한 개인 재무 자동화의 가능성

**요약:** 앤스로픽의 Claude Code와 MCP(Model Context Protocol) 커넥터를 활용해 복잡한 금융 계좌 관리를 자동화하려는 시도가 주목받고 있습니다. 기존에는 웹 크롤링이나 CLI 기반의 크론 잡(cron-job) 방식을 사용했으나, 이는 2단계 인증(2FA), 패스키(Passkey), 브라우저 렌더링 문제 등으로 인해 자주 오류가 발생하곤 했습니다. 하지만 금융 데이터에 직접 연결된 MCP 커넥터를 사용하면, 사용자는 간단한 프롬프트 명령만으로 여러 계좌의 잔액, 거래 내역, 투자 및 대출 상태를 실시간으로 확인하고 반복적인 재무 점검 루틴을 자동화할 수 있습니다.

**의미:** 이는 'AI 에이전트'가 단순히 정보를 찾는 수준을 넘어 사용자의 실제 자산과 데이터를 안전하게 다루는 '개인 비서'로 진화하고 있음을 보여줍니다. 특히 기존의 경직된 자동화 도구들이 해결하지 못했던 보안 및 인터페이스 문제를 AI 커넥터 기술로 극복하면서, 누구나 자신만의 정교한 금융 관리 시스템을 구축할 수 있는 시대가 열리고 있습니다.

## 3. grdpwasm: 플러그인 없는 웹 기반 RDP 클라이언트 등장

**요약:** 별도의 플러그인이나 소프트웨어 설치 없이 웹 브라우저만으로 Windows 원격 데스크톱(RDP)에 접속할 수 있게 해주는 'grdpwasm' 프로젝트가 공개되었습니다. 이 도구는 Go 언어로 작성된 WebAssembly(Wasm)를 브라우저에서 실행하고, 서버 측의 WebSocket-to-TCP 프록시를 통해 통신합니다. 브라우저가 직접 열 수 없는 RDP의 TCP 연결 문제를 서버 측 프록시가 해결해 주는 구조로 작동하며, 이를 통해 완전한 웹 기반의 원격 제어 환경을 구현했습니다.

**의미:** WebAssembly의 강력한 성능과 호환성을 다시 한번 입증한 사례입니다. 복잡한 네트워크 프로토콜을 웹 브라우저 안으로 가져옴으로써 사용자는 어떤 환경에서든 운영체제의 제약 없이 업무용 PC나 서버에 안전하게 접근할 수 있게 됩니다. 이는 기업의 원격 근무 환경 구축이나 보안 솔루션 설계에 있어 중요한 기술적 대안이 될 것으로 보입니다.

## 4. CSS를 바라보는 새로운 시각: 스타일링 언어에서 질의 언어로

**요약:** CSS의 선택자(Selector)와 규칙(Rule) 구조가 사실은 데이터베이스 질의 언어인 Datalog와 매우 유사하다는 흥미로운 분석이 제기되었습니다. 예를 들어, `div.awesome`과 같은 선택자 결합은 특정 조건을 만족하는 요소들의 '교집합'을 구하는 과정이며, 이는 Datalog에서 같은 변수를 반복하여 조인(Join)을 수행하는 논리적 구조와 맞닿아 있습니다. 단순히 웹페이지를 꾸미는 도구를 넘어, CSS를 DOM이라는 데이터 구조 위에서 특정 집합을 고르고 속성을 적용하는 '선택 및 질의 엔진'으로 해석하는 관점입니다.

**의미:** 프론트엔드 개발자들에게 CSS에 대한 깊이 있는 통찰을 제공합니다. CSS를 단순히 시각적 도구로만 보지 않고 논리적인 질의 언어로 이해할 때, 더 효율적이고 구조적인 스타일시트를 설계할 수 있기 때문입니다. 기술의 본질적인 동작 원리를 탐구함으로써 더 나은 코딩 패러다임을 고민하게 만드는 주제입니다.
